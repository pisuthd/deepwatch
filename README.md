## What is DeepWatch?

DeepWatch turns DeepBook's institutional-grade signals into something a human (or an agent) can act on in seconds.

[DeepBook](https://docs.sui.io/onchain-finance/deepbook-predict/) is Sui's central-limit-order-book exchange and prediction market. Its **Block-Scholes oracle** and **sub-400ms settlement** make it fast and accurate, but the raw signal is dense: SVI surfaces, multi-venue odds, time-decayed strikes. The information exists; the clarity doesn't.

DeepWatch solves that with three things no other DeepBook frontend ships together:

1. **A unified trading terminal** for Spot, Margin, and Predict in one UI — pick a product from the sidebar, trade, and never lose context.
2. **AI insights**, generated by Claude 4.6 / MiniMax M3 from SVI oracle data + Polymarket + Kalshi odds, written to **Walrus** and **encrypted with Seal** so the gate is the protocol, not a server.
3. **A stake-to-access pool** (Move smart contracts on Sui) that mints a non-transferable `Subscription` NFT and unlocks the encrypted slice — pro-rata yield for stakers, time-bounded access, on-chain expiry.

Everything you see on the dapp — every signal, every encrypted payload, every access decision — is verifiable on Walrus or Sui.

## Why it matters

| The problem | What DeepWatch does about it |
| --- | --- |
| AI trading insights are usually hosted on a server — opaque, censored, single-tenant. | **Walrus + Seal**: insights live on decentralized storage; the encrypted slice is gated by `seal_approve` on a Move `Subscription` NFT. No middleman. |
| Prediction-market odds disagree across venues (Polymarket, Kalshi, DeepBook). When they disagree, there's an edge. But cross-venue spread is hard to read at a glance. | **Compare table** anchors every DeepBook oracle to its closest Polymarket + Kalshi match by expiry and shows the consensus median in a tooltip. |
| Staking-as-access usually means the access right can be sold separately from the underlying LP — and it always is. | **Non-transferable `Subscription` NFT** (`key + store`, no `transfer`). The NFT dies with the stake. |
| Tatum/Walrus credits are a recurring cost; sometimes the API is down and the dapp is dead. | **Local + Walrus insight source toggle**: the dapp stays useful even if Walrus upload is offline, with the same SSE consumer, the same insight shape, the same UI. |
| Auto-trading is risky when the agent has no opinion on quality. | **Auto Trade** allocates a budget only across markets where the AI returned a non-`NEUTRAL` signal with confidence above the threshold. |

## How it works

### Architecture

```
                           ┌──────────────────────────┐
                           │      Browser (React)     │
                           │  Next.js 16 · React 19   │
                           └────────────┬─────────────┘
                                        │
        ┌───────────────────────────────┼───────────────────────────────┐
        │                               │                               │
        ▼                               ▼                               ▼
 ┌──────────────┐               ┌──────────────┐                ┌──────────────┐
 │  Sui gRPC    │               │  Tatum API   │                │  Walrus +    │
 │  DeepBook V3 │               │  Polymarket  │                │  Seal key    │
 │  Spot/Margin │               │  Kalshi      │                │  servers     │
 │  Predict     │               │              │                │  (Walrus TEE)│
 └──────┬───────┘               └──────┬───────┘                └──────┬───────┘
        │                              │                               │
        ▼                              ▼                               ▼
 ┌──────────────────────────────────────────────────────────────────────────┐
 │                    AI insight generator (MiniMax M3)                     │
 │     SVI oracle + cross-venue odds → directional signal + confidence      │
 │                          → BatchInsight { results, encrypted tail }     │
 └──────────────────────────────────────────────────────────────────────────┘
                                        │
                                        ▼
                       ┌────────────────────────────┐
                       │     On-chain (Sui Move)    │
                       │  deepwatch::pool           │
                       │  deepwatch::subscription   │
                       │  PLP stake + lend          │
                       │  seal_approve gate         │
                       └────────────────────────────┘
```

### The data flow in three steps

**1. Snapshot.** Every 90 seconds the Compare page pulls live DeepBook oracles (Spot / Margin / Predict) and pairs each one with its closest Polymarket + Kalshi match by expiry. The snapshot is the AI batch.

**2. Generate.** The batch is streamed through an SSE consumer (`/api/insights/generate`) backed by MiniMax M3 (Anthropic-compatible). Each market gets a `MatchAnalysis`: signal (`UP` / `DOWN` / `NEUTRAL`), confidence, position size, reasoning, optional macro take.

**3. Persist with access control.** The full set is wrapped with Seal (AES-256-GCM bulk payload, Seal-wrapped AES key under the pool's namespace). The first 6 markets stay plaintext as a free "preview slice" — anyone can read them, including non-stakers, so the AI's directional read is public. The encrypted slice is only recoverable by a wallet holding an unexpired `Subscription` NFT for the matching pool.

### The on-chain access gate

```
User wallet (Sui)                Subscription NFT              Seal key server
        │                              │                              │
        │  holds Subscription          │                              │
        │  ─────────────────────► seal_approve(id, &sub, &clk)        │
        │                              │                              │
        │                              ├──► pool_id prefix check       │
        │                              ├──► clock < expires_at_ms      │
        │                              ├──► sender == sub.owner        │
        │                              │                              │
        │                              │◄─── key share (TEE-attested) ─┤
        │                              │                              │
        │  AES-decrypt full payload ◄──┤                              │
        │                              │                              │
```

The `seal_approve` entry function lives in [`contracts/sources/subscription.move`](contracts/sources/subscription.move) and is called by the Seal key server inside a simulated PTB — the key server refuses to release its share if the PTB aborts.

## Quick start

Prereqs: Node 20+, a Sui wallet (Slush, Sui Wallet, or any wallet-standard-compatible extension), and a Tatum API key (free tier works for Walrus upload/list; Polymarket + Kalshi search are public endpoints).

```bash
git clone <your fork url>
cd deepwatch/frontend
npm install
cp .env.example .env       # then fill in your keys — see the table below
npm run dev
```

Open `http://localhost:3000`. The dev server uses HMR.

### Environment variables

| Var | Purpose |
| --- | --- |
| `NEXT_PUBLIC_TESTNET_GRPC` | Sui testnet gRPC endpoint (Tatum gateway) |
| `NEXT_PUBLIC_MAINNET_GRPC` | Sui mainnet gRPC endpoint (Tatum gateway) |
| `NEXT_PUBLIC_TATUM_API_KEY` | Tatum API key — used for Walrus upload/list and Polymarket/Kalshi search |
| `MINIMAX_API_KEY` | MiniMax Anthropic-compatible API key (insight generation) |
| `MINIMAX_BASE_URL` | MiniMax base URL — override only if you proxy |
| `MINIMAX_MODEL` | MiniMax model id. Default: `MiniMax-M3` |
| `MINIMAX_THINKING_BUDGET` | Optional extended-thinking budget in tokens. Set to `0` to disable. Default: `2048` |

Notes:
- `.env` is gitignored; the committed template is [`frontend/.env.example`](frontend/.env.example).
- Tatum gRPC URLs and `NEXT_PUBLIC_TATUM_API_KEY` ship to the browser by design — Tatum is CORS-allowed for client-direct calls.
- `MINIMAX_*` env vars must **never** be `NEXT_PUBLIC_*`. They are consumed only inside [`frontend/app/api/insights/generate/route.ts`](frontend/app/api/insights/generate/route.ts).

### Run the contracts (Sui testnet)

```bash
cd contracts
sui move build
sui move test
sui client publish --gas-budget 200000000
```

The first publish creates the `PoolCap` (one-shot, used by the demo admin functions). Subsequent `pool::init` calls create a fresh `Pool<PLP, SUI>` and bind it to the dapp's `NEXT_PUBLIC_DEEPWATCH_POOL_OBJECT_ID`.

## Roadmap

The dapp is **live on mainnet (Spot) and testnet (Spot, Margin, Predict)** today and runs end-to-end.

**Next up:** A downloadable AI trading agent. The agent will read insights shared on Walrus and use them — combined with a locally fine-tuned model — to trade autonomously on DeepBook and beyond. The plumbing for this is already partly in place: every AI batch is signed, timestamped, and published on Walrus with the encrypted slice gated by the on-chain `Subscription` — exactly the verifiable signal an autonomous agent needs to trust a third-party insight.
